[retro] 2026-02-13 (build fix + QB v4 spec)

症状
- `npm run build` が TypeScript で落ちる:
  - `style_copy_dictionary.ts:1:1 Invalid character`（先頭に `\` が混入）
  - または `: expected`（TS 内に `#region` が混入）

原因
- PowerShell/zip overlay 由来で TS ファイル先頭に余計な文字が入った
- PowerShell の `#region` を TS にそのまま貼ってしまった（TS では無効）

対応
- TS 先頭の余計な `\` を削除
- `#region/#endregion` 行を `//` コメント化
- UTF-8 (no BOM) で書き戻し

副作用/注意
- Next が `C:\Users\User\package-lock.json` を workspace root と誤認する警告が出る
  - 安全に .bak へ退避推奨（再発防止）

成果物
- src/oshihapi/modes/style_copy_dictionary.ts（ビルドできる版）
- docs/question_bank_v4_spec_latest.md（分岐構造の最新仕様）
