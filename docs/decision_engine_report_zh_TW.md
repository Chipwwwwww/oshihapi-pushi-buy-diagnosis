# オシハピ 決策診斷設計（繁體中文版）

## 前言

推し活粉絲在「要不要買」上的判斷，表面上很像新古典經濟學的效用最大化，但實務上會被 **FOMO（錯過焦慮／取り残され不安）**、對後悔的恐懼、預算約束、稀缺性等心理因素強烈牽引。研究指出，突發的衝動購買常由強烈情緒需求引發，並會在現場環境、促銷刺激、個人滿足感與自我評價等因素疊加時顯著增強【211056856343116†L217-L228】。SNS 放大的 FOMO 會促進衝動購買與購後後悔【534064009337453†L60-L67】；稀缺性訴求在看到他人互動／入手稀缺商品時，會引發心理反抗（reactance）並進一步放大 FOMO【127802982918067†L425-L433】。另外，對「價格／缺貨」的後悔敏感度會影響購買意向：例如高價先行預約後遇到降價，或延後購買結果缺貨，都會被納入消費者的預期後悔計算【149926660455017†L203-L211】。

因此，把這些心理與經濟因素「參數化」、並用少量題目快速估計，就能做出 **可解釋、可調參、而且適合推活現場使用** 的購買判斷工具。

以下提出三種流程：

- **短診斷（30 秒）**：4–5 題，現場秒判。
- **中間診斷（約 2 分鐘）**：6–8 題，加入轉售、收藏完成度、保管負擔等中長期因素，並採用少量交叉項與非線性項強化極端情境。
- **長診斷（AI 後續評估指令生成）**：10–12 題 + 自由輸入，生成可餵給 AI 的結構化指令，協助使用者做更深入的後續評估與行動規劃。

同時也會提供：每個題目對應的參數、交叉項／非線性項、以及對應的「理由模板」。

## 設計用參數（Parameters）

各題會估計下列參數（可能一題對多參數）。數值建議使用 **0–5**（5 代表更強），之後統一縮放到 **0–1** 用於計分。

| 參數符號 | 含義 | 角色（設計意圖） |
|---|---|---|
| **α 推し度／情熱（Affinity）** | 對推し的愛著、支持欲、情感連結。 | 與稀缺性一起會放大「現在不買就會後悔」的力量（可用交叉項強化）【127802982918067†L425-L433】。 |
| **β 稀缺性（Scarcity / FOMO Trigger）** | 限量、期間限定、熱門程度、再販可能性等「稀缺」認知。 | 稀缺性訴求會放大 FOMO 並誘發衝動反應【127802982918067†L425-L433】；可單獨加分，也可與 α 交叉。 |
| **γ 預算痛（Budget Pain）** | 價格／遠征費用帶來的心理負擔與現金流壓力。 | 與後悔風險交叉時，常是「需要踩剎車」的核心訊號【149926660455017†L203-L211】。建議用非線性（超過門檻後急劇變痛）。 |
| **δ 後悔風險（Regret Risk）** | 「買了會後悔」與「不買會後悔」的預期後悔。 | 研究顯示：上向後悔（怕之後更便宜）會抑制衝動購買；下向後悔（怕之後更貴／缺貨）會促進衝動購買【397078993502703†L365-L404】。可用題目區分方向。 |
| **η 機會成本（Opportunity Cost）** | 同期還有其他推活／支出（演唱會、抽選、同月大坑），是否需要留預算。 | 一般是負向；但可與 α 做交叉，反映「推し優先級」。 |
| **r 轉售／流動性（Resale Liquidity）** | 你覺得這個東西「好不好轉手／好不好賣回」。 | 通常用來緩和 γ、δ：越能轉手，越像是買一個“可回收選項”。 |
| **c コンプリート價值（Completion Value）** | 買了是否能「補齊／完成」系列或收藏。 | 常呈現跳變（非線性）：完成的一刻效用暴增；與 α 交叉更顯著。 |
| **d 日常使用／實用性（Daily Utility）** | 買完是否會常用／常看／常拿出來。 | 越高通常越不後悔；可用來降低 δ，並與 γ 交叉表現「貴但很常用→值得」。 |
| **h 保管／維護負擔（Storage Hassle）** | 置放空間、整理管理、灰塵維護等成本。 | 太多物品會增加壓力與焦慮、帶來認知負荷【652243751849733†L52-L59】【652243751849733†L74-L90】；h 高通常是負向，並可與 α 交叉。 |
| **t 體力／時間成本（Travel Time/Energy）** |（票券／遠征特化）移動時間、請假、體力負擔、行程風險。 | 一般負向；可與 α 交叉表達「推し夠強就願意扛」。 |
| **p 時間壓力（Time Pressure）** | 截止時間、現場倒數、剩餘數量提示等「被催」程度。 | 高 p 會降低冷靜度；與 δ 交叉常用來觸發「先冷卻／再想想」【211056856343116†L217-L228】。 |

## 1) 短診斷（30 秒）

### 目標

短診斷提供「現場秒判」：**4–5 題**抓住最關鍵的四大軸（α、β、γ、δ），再視情況加一題 η。

### 題目清單（建議 UI 為 0–5 量表，或 5 段選項）

> 下表同時附上「日文題目（可直接用於產品）」與「繁中解釋」。如果你要做純日文版 UI，可直接取日文句子。

| No. | 題目（JP） / 說明（繁中） | 估計參數 |
|---|---|---|
| Q1 | **JP：**「推しへの愛や応援したい気持ちはどれくらいありますか？」（0–5）  
**繁中：**你對推し的喜愛／支持衝動有多強？ | α |
| Q2 | **JP：**「この商品/チケットはどのくらい限定・希少だと思いますか？」（0–5）  
**繁中：**你覺得它的稀缺性／限量程度有多高？ | β |
| Q3 | **JP：**「この価格や遠征費用はお財布にとってどれくらい負担ですか？」（0–5）  
**繁中：**這個價格（含遠征費）對你有多痛？ | γ |
| Q4 | **JP：**「今買わずに後で後悔しそう？それとも買った後に後悔しそう？」（0–5）  
**繁中：**你比較怕「不買會後悔」還是「買了會後悔」？（可做成方向題＋強度題） | δ（可含方向） |
| Q5（可選） | **JP：**「今月、他の推し活や大きな出費と重なっていますか？」（0–5）  
**繁中：**同月還有其他大坑／重要活動會吃掉預算嗎？ | η |

### 判定邏輯（短診斷版）

- 只使用 **線性項 + 1–2 個交叉項**（避免理由變複雜）。
- 典型交叉：
  - **αβ** 高 →「推し很強 + 超限定」：更偏向 *買*。
  - **γδ** 高 →「很痛 + 很怕後悔」：更偏向 *再想想／不買*。

> 短診斷不建議上非線性（例如平方），因為你要的是「直覺易懂」與「一句話理由」。

## 2) 中間診斷（約 2 分鐘）

### 目標

中間診斷提供「更可信的理由」：維持 **6–8 題**（周邊與票券略有差異），加入能大幅改變決策的中長期因素：轉售流動性、コンプ完成度、日常使用、保管負擔、（票券）遠征成本、時間壓力。

### 題目清單（建議做成卡片式：每題 5–10 秒）

| No. | 題目（JP） / 說明（繁中） | 估計參數 |
|---|---|---|
| Q1 | 短診斷 Q1：推し度 | α |
| Q2 | 短診斷 Q2：稀缺性 | β |
| Q3 | 短診斷 Q3：預算痛 | γ |
| Q4 | 短診斷 Q4：後悔風險（可含方向） | δ |
| Q5 | **JP：**「メルカリなどで売ったり譲ったりすることに抵抗はありますか？」（0–5）  
**繁中：**你對轉售／二手（非黃牛）是否排斥？（越不排斥越能回收風險） | r |
| Q6 | **JP：**「これを買うと、シリーズ/コレクションが完成しますか？」（0–5）  
**繁中：**買了能補齊一套／完成系列嗎？ | c |
| Q7 | **JP：**「買った後、日常でどれくらい使う/眺めると思いますか？」（0–5）  
**繁中：**買完你會常用／常看嗎？ | d |
| Q8 | **JP：**「置き場所・管理の負担はどの程度ですか？」（0–5）  
**繁中：**你現在的空間／管理負擔大嗎？ | h |
| Q9（票券用） | **JP：**「移動・宿泊・休みの取りやすさなど、体力/時間の負担は？」（0–5）  
**繁中：**遠征時間／體力／請假難度會不會很硬？ | t |
| Q10（可選） | **JP：**「締切が迫っていて焦っていますか？」（0–5）  
**繁中：**截止時間/現場倒數帶來的焦躁感有多強？ | p |

> **周邊流程**：通常不需要 Q9。  
> **票券流程**：建議保留 Q9；Q10 視場景（抽選/一般販售/現場）決定。

### 中間診斷：推薦的交叉項與非線性項（少而精）

中間診斷的「準」不是靠高階多項式，而是靠**少量可解釋的放大器**。

#### 交叉項（Interaction Terms）

1) **αβ（推し度 × 稀缺性）**：
- 推し很強且超限定時，FOMO 更容易被放大【127802982918067†L425-L433】 → 推向「買」。

2) **γδ（預算痛 × 後悔風險）**：
- 「很痛又很怕後悔」是最典型的剎車訊號 → 推向「再想想／不買」。

3) **rγ、rδ（流動性 × 預算痛/後悔）**：
- 越能轉手，越能緩和「痛」與「怕後悔」。

4) **cα（完成度 × 推し度）**：
- コンプ完成對高推し族是效用跳變（非線性），因此用交叉項放大。

5) **pδ（時間壓力 × 後悔）**：
- 越急越容易衝動、也更容易後悔；衝動購買在認知資源不足時更常見【211056856343116†L217-L228】。

#### 非線性項（Nonlinear Terms）

6) **預算痛的門檻平方：\((\max(0,\gamma-\gamma_0))^2\)**
- 低於門檻痛感緩；一旦超過某個「心理可承受線」就急遽惡化。

7) **稀缺性平方：\(\beta^2\)**
- 普通限定只是加分；「幾乎完售／真的不會再來」時會爆發式提高購買衝動。

### 理由模板（中間診斷用）

> 重點：每個模板要能對應到「觸發條件」，並能用朋友式語氣說清楚原因。

| 觸發（例） | 理由模板（繁中；可再做日文化） |
|---|---|
| **αβ 高** | 「你對推し的愛很強，而且這次又很限定。看到別人入手時 FOMO 很容易被放大【127802982918067†L425-L433】——如果你覺得錯過會很痛，現在入手能換到更高的安心感。」 |
| **γδ 高** | 「這筆錢對你來說真的有點痛，加上你也覺得『買了可能會後悔』。這種組合很容易讓心情被拉扯，建議先冷卻一下，把上限/替代方案想清楚再決定【149926660455017†L203-L211】。」 |
| **rγ 或 rδ 高（=可轉手）** | 「即使之後想改變主意，你也不排斥走二手/轉讓路線，代表風險有出口。對於偏高的價格或不確定性，這會讓你更有餘裕『先買再調整』。」 |
| **cα 高（=買了就コンプ）** | 「這次買下去就能補齊/完成系列，而且你又是高推し。完成一套的滿足感通常是跳上去的——如果你一直在等這塊拼圖，這筆支出很可能是『花得值得』的那種。」 |
| **pδ 高（=很急又很怕後悔）** | 「你現在有點被時間推著走，又很擔心後悔。急迫感會讓人更難冷靜【211056856343116†L217-L228】。如果不是“錯過就會崩”的程度，建議先做 10 分鐘冷卻或設提醒，再回來看一次結果。」 |
| **h 高（=空間/管理壓力大）** | 「你目前的空間/管理負擔已經偏高。環境雜亂會增加壓力與認知負荷【652243751849733†L52-L59】【652243751849733†L74-L90】——如果買回來會讓你更累，那就算很喜歡也可能變成後悔來源。」 |

## 3) 長診斷（生成可餵給 AI 的後續評估指令）

### 目標

長診斷不是只為了判定「買/不買」，而是要把使用者的情境**結構化**，產生一段 **可直接丟給 AI** 的指令（prompt），請 AI 做更深入的：

- 風險情境拆解（會後悔的路徑有哪些？）
- 行動計畫（冷卻、查價、轉售、預算排程、遠征規劃）
- 情緒照顧（FOMO 安撫、合理化與界線設定）

### 追加題目（在中間診斷 Q1–Q10 基礎上）

| No. | 題目（JP） / 說明（繁中） | 產出 |
|---|---|---|
| Q11 | **JP：**「商品/チケットの名称・種類を入力してください。」  
**繁中：**輸入商品/票券名稱與類型（手辦、盲抽、套組、座位等） | 給 AI 的上下文 |
| Q12 | **JP：**「これを通じて得たい体験・目的は何ですか？」（自由輸入）  
**繁中：**你想透過這次購買得到什麼？（支持推し、紀念、社交、收藏、投資等） | 目的／價值觀 |
| Q13 | **JP：**「似た買い物で後悔した/満足した経験は？」（自由輸入）  
**繁中：**過去類似購買：後悔/成功案例是什麼？ | 調整 δ（質性校正） |
| Q14 | **JP：**「買った後の保管/活用プランは？」（自由輸入）  
**繁中：**買回去你打算怎麼放／怎麼用？ | 校正 d、h（計畫性） |
| Q15（票券） | **JP：**「遠征先・日程・他イベントとの兼ね合いは？」（自由輸入）  
**繁中：**遠征地點/日程/與其他活動衝突如何？ | 補充 t、η |

### AI 指令輸出格式（可直接用）

你可以在前端把所有答案組成 JSON，或用以下「人類可讀」格式直接塞給 AI。

### 長診斷可追加的交叉／非線性候補（但要注意可解釋性）

長診斷可以再加一些細緻的交互作用，但建議 **先做“稀疏”**（只保留少數高價值的交叉），避免你自己調參地獄。

- **αη**：同月很多坑，但推し很高 → 仍可能優先（也可反向提醒「留一點給下個更重要的」）。
- **cδ**：想補齊但又怕後悔 → 典型糾結；適合觸發「先查二手價/再販消息」行動。
- **dγ**：很常用就比較不怕貴；把 d 當作 γ 的緩和項。
- **hα**：推し很高但空間爆炸 → 容易買了反而壓力更大；適合觸發「先清空空間」的建議。
- **tα**：遠征很累但推し很高 → 看“你願意為推し扛到什麼程度”。

## 4. 以使用者回饋驅動的機器學習調參計畫（重點：不增加診斷摩擦）

你現有的引擎是「可解釋的加權規則」，非常適合做成：

> **規則可解釋（前台不變）＋參數可學習（離線更新權重）**

關鍵不是把題目變多，而是用**結果頁 1 點回報**＋**下次開啟的輕量回顧**，拿到 ML 需要的 label。

### 4.1 最小必收 label（用於訓練/校正）

這些不放進問卷流程，全部放在結果頁/歷史頁，避免拖慢 30 秒與 2 分鐘診斷。

#### L1. 行動結果（結果頁 1 點）

**JP UI 文案：**

- 「このあとどうした？」
  - 買った
  - いったん保留（再想）
  - 買わなかった
  - まだわからない（後で）

**用途：**最小教師信號（使用者實際行為），用來校正 buy/wait/no 的門檻與權重。

#### L2. 滿意/後悔（下次開啟 or 歷史頁「振り返る」）

**JP UI 文案：**

- 「買ってよかった度」0〜5
- 「後悔」0〜5

研究指出，決策前的預期後悔（anticipated regret）會影響猶豫、懷疑與衝動購買行為【397078993502703†L365-L404】。

**用途：**你的核心價值是「降低後悔」，所以 L2 是最關鍵的目標函數。

#### L3. 金錢痛感（L2 同時）

**JP UI 文案：**

- 「金銭的にしんどかった？」0〜5

**用途：**把「買了也開心但荷包爆炸」分離出來，避免模型只追求“買”。

### 4.2 不增加題數的“自動行為訊號”（用作弱特徵）

你提到的「每題回答時間、回答習慣」很值得收，但把它當作**弱特徵**：
它能近似“迷惘/不確定度”與“衝動/認知資源不足”。研究也提到衝動購買會在情緒刺激疊加、認知資源下降時更容易發生【211056856343116†L217-L228】。

**建議欄位（可先只存本機，若有同意才上傳）：**

- `time_total_ms`
- `time_per_q_ms[]`
- `num_changes`（改答案次數）
- `num_backtracks`（返回次數）
- `abandon_step`（做到哪一題離開）
- `actions_clicked`（官網/中古搜尋/分享/複製）

### 4.3 學習目標（label）怎麼設計

建議同時保留 3 種 label（資料量不同也能逐步上線）：

- **y_action（多類）**：買了/保留/沒買（L1）
- **y_regret（回歸或二值）**：後悔 0–5（L2）
- **y_finance_pain（回歸）**：金錢痛 0–5（L3）

最後輸出仍是 buy/wait/no，但可以用「預測後悔/金錢痛」來拉高 wait 的比例，實現產品價值（少後悔）。

### 4.4 推薦模型：先從“係數可回寫”開始

因為你需要可解釋理由（reason templates），最適合從：

1) **多類 Logistic Regression（softmax）＋ L1（稀疏化）**
   - 特徵：α…p + 候選交叉項（αβ、γδ、cα、pδ…）+ 少量非線性展開（β^2、max(0,γ-γ0)^2）
   - L1 會把沒用的交叉項係數壓到 0，等於自動選特徵

2) （下一階段）**GAM/EBM**：更自然地學門檻形狀，但工程稍重

### 4.5 部署方式：推論仍在前端、訓練離線做

- 前端：仍使用你現在的引擎（加總/門檻/理由規則），只更新 `engineConfig.ts` 的係數
- 後端/本機訓練：每週/每兩週離線訓練 → 匯出係數 JSON → 發佈

推論成本幾乎只有乘加運算，不會影響“快速診斷”。

### 4.7 ML 與理由模板怎麼接（避免黑盒）

即使係數被 ML 更新，前端仍以“發火規則”選理由：

- `alpha*beta` 高 → 「推し×限定導致 FOMO 強」
- `gamma*delta` 高 → 「很貴＋後悔不安高，建議先冷卻」
- `(max(0, gamma-gamma0))^2` 強 → 「超過預算線，痛感急速上升」

只要維持 **“特徵 → 理由模板”** 的對照表，ML 只會變成「調權重」而不是「取代可解釋性」。

## 結語

透過 **短診斷 → 中間診斷 → 長診斷（AI 指令）** 的分層設計，你可以同時滿足：

- 現場要快（30 秒）
- 要有理由、要能說服自己（2 分鐘）
- 想更深挖情境與行動（長診斷 + AI）

整體設計背後的核心是：衝動購買會在強烈情緒與 FOMO 下被放大，尤其當認知資源下降（疲勞、倒數壓力）時更容易發生【211056856343116†L217-L228】；而價格/缺貨的預期後悔會影響購買與等待策略【149926660455017†L203-L211】。同時，生活空間的雜亂與物品負擔也可能造成壓力與焦慮【652243751849733†L52-L59】【652243751849733†L74-L90】——所以把「保管負擔」納入，能讓診斷更貼近真實的推活生活。
