Implementation (minimal diff)
A) Create formatter helper:
   - Create: src/oshihapi/modes/formatResultByMode.ts
   - Input: { runId, verdict, waitType?, reasons[], reasonTags[], actions[], mode, stickerOffset? }
   - Output: { sticker, shareTextX280, shareTextDmShort }
   - Steps:
     1) Determine primaryTag by scanning MODE_PRIORITY_TAGS[verdict] against reasonTags.
     2) Resolve scenarioKey with SCENARIO_RESOLUTION.resolve(verdict, waitType, primaryTag).
     3) Pick sticker deterministically from MODE_DICTIONARY[mode].stickers[scenarioKey]:
        idx=(hash(runId)+(stickerOffset??0))%pool.length
     4) Build share text from MODE_DICTIONARY[mode].text.templates (x_280, dm_short), replacing:
        {verdict} {waitType} {reasons} {actions} {sticker} {emoji} {kaomoji}
        - For BUY/SKIP, waitType should be empty string.
     5) Enforce constraints:
        - emoji/kaomoji limits and allowed lists
        - forbiddenSubstrings: if triggered, fallback sticker to MODE_DICTIONARY.standard (same scenarioKey) and strip disallowed tokens

B) Wire UI

   - Persist mode selection in localStorage; default "standard"
