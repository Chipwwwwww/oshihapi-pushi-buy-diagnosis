オシハピ｜推し買い診断 結果ページ UI 更新メモ
更新日: 2026-02-08

目的
- 結果ページで「やめる｜保留｜買う」を直感的に見せる（刻度尺＋矢印）
- 理由/推奨行動/共有を “読みやすく・押しやすく” 整理する
- 友達テストで「納得感」と「シェア/再利用」を上げる

追加予定ファイル
1) components/DecisionScale.tsx
- 三分割スケール（やめる/保留/買う）＋赤い矢印で現在位置
- props:
  - decision: "buy"|"wait"|"no"
  - index: -1..+1（0が保留中心）

結果ページでの統合ポイント（app/result/[runId]/page.tsx）
- エンジン出力に「連続値」が無ければ、score/threshold から index を導出する
  - 例: index = clamp((score - mid) / (range/2), -1, 1)
- decision を "buy"/"wait"/"no" に正規化して DecisionScale に渡す
- 画面順:
  1) 結論（大見出し）＋サブテキスト
  2) DecisionScale
  3) 今すぐやる（推奨行動 2〜3個）
  4) 理由（3〜5個、チップやアイコン可）
  5) shareText（コピー＋トースト）
  6) L1フィードバック（買った/保留/買わなかった/まだ）

A/B で見たい指標（将来）
- 共有コピー率（copy click）
- 行動ボタン（公式/中古/相場）クリック率
- 完了率（結果ページ到達率）
- L1: 行動結果（買った/保留/買わなかった）の比率
