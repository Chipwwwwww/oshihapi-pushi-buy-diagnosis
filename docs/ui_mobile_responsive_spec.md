# モバイルUI改善 仕様（Result/History/Flow 共通）

## 目的
スマホ（iPhone Safari）で「読めない・ごちゃつく・余白が変」になっているのを直す。  
MVPとして **視認性（コントラスト/文字サイズ）** と **レイアウト（横スクロール/安全領域）** を最優先。

---

## 1) UI原則（日本市場/若年ACG）
- 口調：友達っぽく、でも整理されている（説教しない）
- 情報の優先順位：
  1. 判定（買う/保留/やめる）
  2. 一言理由（1行）
  3. DecisionScale（傾き/信頼度）
  4. 次のアクション（1〜3）
  5. 理由カード（3〜6）

---

## 2) レスポンシブ基本
### コンテナ
- `max-width: 720px`
- モバイル：左右 16px / 上下 20〜24px
- SM以上：左右 24px / 上下 28〜32px

### タイポグラフィ
- 本文：16px基準、line-height 1.65
- 見出し：`clamp()`でスケール
  - H1: `clamp(28px, 6vw, 44px)`
  - H2: `clamp(18px, 4.2vw, 24px)`

### コントラスト（必須）
- 暗背景で `muted` が暗すぎる事故を防ぐ（muted は“薄いが読める”が最適）
- カード背景（surface）と文字（text）のコントラストを担保

### 安全領域
- 下：`padding-bottom: calc(24px + env(safe-area-inset-bottom));`
- `viewport-fit=cover` を前提に崩れない

---

## 3) DecisionScale（スマホ最適）
- ラベル「やめる/保留/買う」は 12–14px 以上
- マーカーは必ず見える（アクセント色/太さ/サイズ）
- 420px 以下では2段化を許容（スケール→ラベル）

---

## 4) 触りやすさ（iOS）
- ボタン高さ：44px以上
- タップ領域に余白（縦 10〜12px 以上）
- `:focus-visible` 対応

---

## 5) 受け入れチェック（スクショでOK）
- iPhone幅で横スクロールなし
- 「信頼度」「判定の傾き」が暗背景でも読める
- DecisionScaleのラベル・マーカーが読める
- 下部ホームバーに被らない
